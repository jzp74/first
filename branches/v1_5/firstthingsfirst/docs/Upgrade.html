First Things First - Upgrade

Please follow these instructions and you'll have First Things First upgraded in no time.

Upgrade procedure

  • Download the latest release of FirstThingsFirst (zip format)
  • Dowload your 'localsettings.php' from your hosting provider.
  • Backup your database (very important!)
  • Upload the 'firstthingsfirst' directory from the zipfile to your hosting provider.
  • Upload your local copy of 'localsettings.php' to your hosting provider. Make sure that you add new settings to your version of 'localsettings.php'. Refer to the paragraph below to see if new settings have been added to the 'localsettings.php' of your new version.
  • Run the release specific upgrade script (if present). You can find this script in the 'scripts' directory. Refer to the paragraph below to see if you need to run an update script for your new version.
  • Open a browser and go the webadress of the location to which you copied the application. You will get a login screen. Login as user 'admin' with your admin password.
  • If login is succesful, chances are that your installation was succesful. I recommend you to do the following just to be sure:
    • Go to the following URL: http://www.YOURDOMAIN/LOCATION_OF_FIRSTTHINGSFIRST/test
    • You are presented with the testresults. If they are all green, your installation was definitely a success. If not, the problem usually has something to do with your database settings.

That's all!

As you can see this was really simple. Now you can start creating and using all kinds of lists. Have fun!

Upgrade scripts

Upgrade from version 1.4 to version 1.5

  • Run upgrade script from following URL:
    http://www.YOURDOMAIN/LOCATION_OF_FIRSTTHINGSFIRST/scripts/update_v1.4_to_v1.5.php
  • Add the following line (right under the line with LANG_NL) to 'localsettings.php' (you can also copy them from the new 'localsettings.php' that comes with your new version):
    # LANG_ES - Spanish language

Upgrade from version 1.2 to version 1.3

  • Run upgrade script from following URL:
    http://www.YOURDOMAIN/LOCATION_OF_FIRSTTHINGSFIRST/scripts/update_v1.2_to_v1.3.php
  • Add the following lines to 'localsettings.php' (you can also copy them from the new 'localsettings.php' that comes with your new version):
    # themes
    # choose between the following themes:
    # THEME_BLUE - Original blues
    # THEME_BROWN - Ancient writing
    $firstthingsfirst_theme = THEME_BROWN;

Upgrade from version 1.1 to version 1.2

  • Run upgrade script from following URL:
    http://www.YOURDOMAIN/LOCATION_OF_FIRSTTHINGSFIRST/scripts/update_v1.1_to_v1.2.php
  • Remove the following lines from 'localsettings.php':
    # date setting for all lists
    # choose between DATE_FORMAT_EU and DATE_FORMAT_US
    $firstthingsfirst_date_string = DATE_FORMAT_EU;
  • Remove the following lines from 'localsettings.php':
    # maximum numbers of entries per page
    $firstthingsfirst_list_page_entries = 14;

Upgrade from version 1.0 to version 1.1

  • Run upgrade script from following URL:
    http://www.YOURDOMAIN/LOCATION_OF_FIRSTTHINGSFIRST/scripts/update_v1.0_to_v1.1.php
  • Remove the following lines from 'localsettings.php':
    # full path name to portal on host
    $firstthingsfirst_full_pathname = "d:///sites/firstthingsfirst";
  • Add the following lines to 'localsettings.php' (you can also copy them from the new 'localsettings.php' that comes with your new version):
    # session time in minutes
    $firstthingsfirst_session_time = 30;
  • Remove the lines concerning the logfile from 'localsettings.php' and add the following lines (you can also copy them from the new 'localsettings.php' that comes with your new version):
    # logfile, all logmessages will be written in this file in the ./uploads directory
    # you can add strftime styled parameters, see http://www.php.net/manual/en/function.strftime.php for detailed information
    $firstthingsfirst_logfile = "firstthingsfirst-%Y-%W.log";
